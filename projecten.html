CSS to the Rescue en Browser Technologie

CSS to the Rescue

Tijdens de Vakken CSS to the Rescue en Browser Technologie heb ik twee projecten moeten maken. Voor CSS to the rescue had ik gekozen voor een console. Mijn idee hiervoor was een spel van een kermis. Mijn spel de "Duck Shooter" is volledig met html en CSS gemaakt.

Bij dit vak heb ik heel veel van CSS geleerd. Vooral animaties in CSS heb ik veel mee gewerkt en begrijp ik nu veel beter. Ook heb ik geleerd dat van testen met andere mensen je heel goed verbeterpunten ontdekt, wat goed is voor aankomende projecten.

Browser Technologies

Voor browser technologies was de opdracht om een online belastingaangifte formulier te maken in de huisstijl van de NS. In deze opdracht moest ik opeens veel gebruik maken van Javascript, wat in het begin erg lastig was omdat ik daar nog helemaal niet mee had gewerkt, maar dat hielp wel om dingen zoals eventlisteners en queryselectors te leren begrijpen en gebruiken en dat was heel handig. Ook heb ik hierbij met localStorage leren werken, wat later goed van pas kan werken.

Bij dit vak heb ik meer met Javascript om leren gaan en kan ik het veel beter lezen dan eerst. Ook heb ik veel geleerd over accesibility op het web. Ik snap nu hoe belangrijk het is dat de html van een site goed is opgebouwd, omdat deze sterke basis ervoor zorgt dat alles eromheen ook goed werkt.

Hackaton

Opdracht

Het project voor de hackaton was om een ode te maken aan de belangrijkste vrouwen in tech. Ons groepje heeft dit gedaan, door een podium te maken voor een prijsuitreiking, waarbij de vrouw die geselecteerd wordt, het podium opkomt en wat zij bereikt hebben in beeld komt.

Bij de hackaton heb ik vooral gewerkt met animaties, waar ik tijdens CSS mee geoefend had en nu dus mee door ging.
Dit was het eerste project van de minor waarbij wij in een groep samenwerkten. Dit ging wel goed. Iedereen deed wat er van hun verwacht werd en de rollen waren verdeeld voor iedereens sterkte punten. Qua ontwerp ging het een beetje stroef in het begin, maar toen we uiteindelijk Een concept hadden, ging het heel snel en kwam er een mooi resultaat uit.

Human Centered Design en API

Human Centered Desing

Voor HCD moest er een opdracht gemaakt worden die heel specefiek voor één persoon zou werken. In mijn geval was de opdracht om een toetsenbord te maken voor Erik. Erik heeft namelijk een motorische beperking waardoor zijn spieren niet goed zijn ontwikeld. Het probleem dat Erik had, was dat als hij in de trein berichten wilde sturen, dat vaak niet lukte, omdat de trein ervoor zorgt dat zijn rolstoel veel beweegt en hij daardoor niet goed kan typen.
Voor API moesten we een opdracht maken waarbij we een vaker voorkomende app-concept te maken met behulp van een content API en twee web API's.

Net als bij BT heb ik met dit vak veel met Javascript gewerkt en heb ik het nog beter leren begrijpen. Ik kan nu zelf heel veel javacript schrijven. Wat ook belangrijk bij dit vak was, was het testen en de resultaten gebruiken om het product te verbeteren.

API

Voor API heb ik dus een app gemaakt waarop je scores van en statistieken van wedstrijden kunt zien, om zo te bepalen wat een goeie weddenschap zou zijn. Dit ging in het begin erg moeizaam, omdat ik nog nooit met een API had gewerkt en dus niet wist hoe ik de data moest ophalen en verwerken, maar na een paar weken oefenen lukte dat steeds beter en toen ging het allemaal heel snel. Ik heb toen ook uit data gegevens kunnen halen en daar nieuwe dingen van kunnen maken, zoals het aantal goals voor en tegen optellen en weergeven.

Dit was voor mij waarschijnlijk een van de meest leerzame vakken, ondanks dat het in het begin helemaal niet goed ging. Ik begrijp nu goed hoe een server werkt voor een site en hoe ik daar data uit kan halen. Ook had ik nog nooit met Liquid gewerkt en begrijp ik dat nu ook.

Als ik nog langer de tijd had voor dit project, had ik nog verder willen werken om het concept dat ik had bedacht beter uit de werken. Het idee was om een app te maken die zou helpen met weddenschappen. Een functie die ik nog misde en graag had willen hebben, was om een lijst met mogelijke weddenschappen te weergeven en aan te geven aan de hand van de beschikbare gegevens of dat een goeie optie is of niet.

Helaas is de pagina niet meer bruikbaar zoals deze hier staat, omdat de API die gebruikt werd betaald was en ik die betaling heb stopgezet.
Klik op het grote vak om het nog werkende deel van de site te zien.

Meesterproef

Opdracht

Voor de meesterproef moesten wij een pagina maken dat een collectie AI kunst zou tentoonstellen. Bij deze kunst zouden bijpassende opdrachten komen die de soort kunst zou uitleggen en de gebruiker zou kunnen gebruiken om een beter begrip te krijgen voor de kunst en AI in zijn geheel.
De site zou gebruikt worden door jonge creatieven die meer willen leren over AI kunst en hoe ze het zelf kunnen maken, maar ook door docenten die de oefeningen zouden willen gebruiken om hun leerlingen met AI te laten oefenen.
